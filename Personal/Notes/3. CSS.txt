# CSS - Cascading Style Sheets

((property: value;))

# How to add CSS

* inline - <html style="background: blue"></html>
	 - 1 element
* internal - <style>
	       html {
	         background: red;
	       }
	     </style>
	   - 1 webpage 
* external - similar format as internal, but in a completely different file
	   - <link rel="stylesheet" href="./styles.css" />
	   -1 website

# CSS Selectors

* element selector - h1/a/p {
		       prop: value;
		     }
* class selector - .class {
		     prop: value;
		   }
		 - grouping element in html file to apply same css rule
* id selector - #id {
	          prop: value;
	        }
	      - diff w/ class: id - 1 element in a html file, class - many elements in a html file
* attr selector - eg. p[draggable="false"] {
		        prop: value;
		      }
* universal selector - * {
		         prop: value;
		       }

# CSS Positioning

* static - HTML default flow
* relative(position: relative;) - relative to default position
* absolute(position: absolute;) - position relative to nearest positioned ancestor or top left corner of webpage
* fixed(position: fixed;) - relative to the top left corner of browser window eg ads on webpage
* Z-index

# CSS Display

* display: block; is default(except <span></span>)
* block - take up entire full width
* inline - into the same line (cant set the width and height of content)
* inline-block - like inline but can set width and height
* none - makes object disappear

# CSS Float 

html: <img...>
      <p>text...</p>
      <footer>foot...</footer>
css: img {
       float: left(or right);
     }
     footer {
       clear: left(or right or both);
     }
* the img text will wrap around the image (reverse L-shape)
* footer will not be affected by float
* only use float when wrapping text around img

# CSS Flexbox

* wrap div inside container, target cotainer and set display to flex (100% width)
* flex dont follow inline, block, inline-block rule.
* inline-flex - makes a block to let other content be in the same line
	      - just enough space for the content
* align content in 1D layout
* eg. to be used to make a navbar

# Flex Direction

(parent)
* flex-direction: row (default on main-axis(x) cross-axis(y))
* flex-direction: column (main-axis(y) cross-axis(x)

(child)
* flex-basis: NNpx - height of main axis (go to child)

# Flex Layout

(parent)
* flex-wrap: nowrap; - continue inline even lacking space (default)
* flex-wrap: wrap; - when lacking space, content continue in a new line
* flex-wrap: wrap-reverse; - same as wrap, but at the opposite side of cross-axis
* justify-content(main-axis): flex-start; - content at the start of the container
* justify-content: flex-end; - content at the end of the container
** other: center/space-between/space-around/space-evenly
* align-item(cross-axis) 
* align-content(only works when flex-wrap: wrap;)
* flex-flow(combi of flex-direction + flex-wrap): column wrap;

* wrap - align-content
* nowrap - align-items

(children)
* order: 0; (default)
* order: NN; - bigger NN, more right
* align-self(cross-axis)

# Flex Sizing

* priority: content width < width < flex-basis < min/max-width
* flex-grow: 1; (is ON)
* flex-shrink: 0; (is OFF)
* flex: 1(grow) 1(shrink) 0(basis(set to 0 for equal width)); = flex: 1; (1(grow) 1(shrink) 0(basis))

# CSS Grid

* To make 2D layout
* Can be combined with other things like flex, bootstrap, float etc
* Flex inside grid and grid inside flex is normal
* .container {
    display: grid;
    grid-template-columns: 1fr 2fr; (1:2 ratio)
    grid-template-row: 1fr 1fr; (1:1 ratio)
    gap: 10px;
  }
* fixed size grid is not responsive (px, pt, rem etx) 
* grid-template: 1fr 1fr(row) / 1fr 1fr(column)
* auto: width(column) - grid will try to fill 100% available space
	height(row) - grid will try to fit the content
* minmax - eg. grid-template-columns: 200px minmax(400px, 800px)
* repeat - eg. grid-template-columns: repeat(2, 200px); = 200px 200px;
* grid-auto-rows/columns: NNpx; (new row/columns that doesnt fit our grid, give NNpx height/width)

# Grid Placement

* grid containers contain grid items contain grid cells
* grid lines separating grid cells
* grid-column(/grid-row) contains(:span 2): + eg. grid-column: NN//span(start) / NN//span(end)
** grid-column-start: 2(grid1 empty, grid start in grid2)
** grid-column-end: -1(the grid will send before entering the row below)
* grid-area: NN(row-start) / NN(column-start) / NN(row-end) / NN(column-end)
** all items need to use grid area if 1 uses grid area
* grid lets a content overlay other content (adjust transparency in color)
* grid-template: NN NN(grid-template-rows) / NN NN(grid-template-columns)
