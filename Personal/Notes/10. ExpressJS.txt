** import express from "express";
   const app = express();
   const port = 3000; //location of server

   app.get("/", (req, res) => {
     res.send("Hello");
   });

   app.listen(port, () => {
     console.log(`Server running on port ${port}.`); //callback
   });
* localhost - dont have a server on the internet, instead want to host our server locally. (localhost:3000(port))
* port - doors inside our computer

HTTP Request
* = hypertext transfer protocol - language for computers to communicate with each other
* get - request resource from server (text, website, data, etc)
* post - sending resource to server
* put - update methods (replacing existing data)
* patch - update methods (patching new data to existing data)
* delete - removes existing resource

* replace node w/ nodemon to stop needing to restart server when theres change
* / - endpoint - localhost:3000/
** /about - localhost:3000/about //show about me page

* HTTP response status code (eg 404 error - example of client side error)
** informational responses 100 - 199 "hold on"
** successful responses 200 - 299 "here you go"
** redirection responses 300 - 399 "go away"
** client error responses 400 - 499 "you fucked up"
** server error responses 500 - 599 "i fucked up"

MIDDLEWARES
* preprocess requests - change aspects of the request or perform various functions before it goes to final routing
* log requests - duration, type, status etc
* authentication purposes
* process errors in the requests
* eg BODY PARSERS - parse incoming request bodies in a middleware before your handlers
* pay attention to the order that you arrange your middleware
